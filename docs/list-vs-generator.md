# 列表推导式 vs 生成器表达式

> 第 3 章「集合与推导式」延伸阅读。两者都是**一次性生成数据**的简洁写法；核心区别：**列表推导式立刻算完、占内存**；**生成器表达式按需算、省内存**。

---

## 1. 语法

| | 括号 | 返回类型 |
|---|------|----------|
| 列表推导式 | `[ ]` | `list` |
| 生成器表达式 | `( )` | `generator` |

```python
list_comp = [x * 2 for x in range(5)]   # [0, 2, 4, 6, 8]
gen_exp   = (x * 2 for x in range(5))   # <generator object ...>
```

---

## 2. 计算时机

### 列表推导式 — 立即求值

循环**一次跑完**，所有结果放进一个 `list`。数据量大时占内存多，极端情况可能 OOM。

### 生成器表达式 — 惰性求值

运行时只返回**生成器对象**，不预先算值。只有 `next()`、`for` 迭代、或传给 `sum()` / `max()` 等「逐个消费」的函数时，才**算下一个、用一个**。

---

## 3. 内存对比（直觉）

```python
# 列表：先造 1000 万个 int 的 list，再 sum → 内存高
squares = [x * x for x in range(10**7)]
total = sum(squares)

# 生成器：sum 边取边加，同一时刻几乎只有一个值在内存里
total = sum(x * x for x in range(10**7))
```

---

## 4. 怎么选

| 场景 | 推荐 |
|------|------|
| 数据量小，要多次用、索引、切片、排序 | **列表推导式** |
| 数据量大、只遍历一次 | **生成器表达式** |
| 作为函数唯一参数 | 生成器可省略一层括号：`max(x for x in nums)` |
| 需要 `len()`、下标访问 | **列表推导式**（生成器不支持） |

---

## 5. 生成器只能用一次

```python
gen = (x for x in [1, 2, 3])
print(list(gen))   # [1, 2, 3]
print(list(gen))   # []  ← 已耗尽
```

列表可以反复遍历。

---

## 6. 类比

| | 列表推导式 | 生成器表达式 |
|---|------------|--------------|
| 生活 | 一次买齐一周菜放进冰箱 | 现摘现吃，不占冰箱 |
| 代价 | 占空间 | 不能「重复掐同一批」 |

---

## 7. 再进阶：生成器函数 `yield`

表达式里只能写简单 `for` / `if`；逻辑复杂时用**生成器函数**：

```python
def my_gen(n: int):
    for i in range(n):
        yield i * 2

g = my_gen(5)
print(list(g))   # [0, 2, 4, 6, 8]
```

惰性与 `(x for x in ...)` 相同，但可写多行、多分支。

---

## 8. 对照总表

| 特性 | 列表推导式 | 生成器表达式 |
|------|------------|--------------|
| 语法 | `[expr for x in iter]` | `(expr for x in iter)` |
| 返回 | `list` | `generator` |
| 计算 | 立即全部 | 按需 |
| 内存 | 高 | 低 |
| 重复使用 | ✅ | ❌（迭代一次） |
| 索引 / 切片 | ✅ | ❌ |
| 典型场景 | 小数据、多次操作 | 大数据、一次遍历 |

---

## 9. 和 Java 的对应

| Python | Java（近似） |
|--------|----------------|
| `[f(x) for x in xs if p(x)]` | `stream().filter().map().collect(toList())` |
| `(f(x) for x in xs)` | `stream()` 不 `collect`，边算边用 |
| `def f(): yield x` | `Iterator` / 自定义 `Stream` |

---

## 10. 实践建议

1. **不确定时先用列表推导式** — 可读、好调试。
2. 处理**大文件、大 range、日志流**时改用生成器或 `yield`。
3. 本仓库 `03_collections/lesson.py` 里的 `[x * 2 for x in nums if x > 0]` 是列表推导式；改成 `(x * 2 for x in nums if x > 0)` 再 `sum(...)` 即生成器用法。
